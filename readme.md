# ESE (Economic Simulation Engine) 用户手册

## 面向经济学家：制度对比实验的数字风洞

核心定位：ESE 不是预测工具，也不是纯粹的博弈论模型。它是一个可复现的、硬件无关的反事实检验平台，专门用于对比集中式计划（Central Planning）与分散式市场（Polycentric Markets）在信息失真、时间延迟和破产约束下的制度韧性。

---

## 快速开始：5 分钟跑通你的第一个实验

### 1. 环境准备

```bash
# 克隆项目（假设已获得代码）
cd ese
pip install -r requirements.txt  # 仅需 numpy, pandas, pyyaml
```

### 2. 准备初始世界（seed_world.db）

这是你的"初始国家"——定义了企业（Firm）、家庭（Household）、商品（Good）和政府（Government）的资产负债表。

**方式一：使用内置脚本生成示例世界（2企/5户/3种商品）：**

```bash
python generate_seed.py
```

**方式二：手动修改 SQLite 数据库**，根据你的实验需求设置初始财富分配、产能和商品结构。

### 3. 配置实验参数（config/default.yaml）

```yaml
seed: 42                    # 随机种子，保证复现性
noise_type: "gaussian"      # 信息噪声类型：none(计划完美信息) / gaussian(市场噪音)
base_collateral_ratio: 0.1  # 基础保证金率
order_expire_ticks: 30      # 订单过期时间（相当于"报价有效期"）
```

### 4. 编写你的"制度策略"（核心实验变量）

ESE 将"计划 vs 市场"的差异交由实验者通过 **4 个策略插槽** 来定义：

- **FirmStrategy**：企业如何定价、生产、投资（模拟企业家行为）。

- **HouseholdStrategy**：家庭如何消费、提供劳动力（模拟消费者行为）。

- **GovernmentStrategy**：政府如何征税、发放福利（模拟财政政策）。

- **AllocationPolicy**（制度灵魂）：如何匹配买卖订单。

  - 写一个 **市场分配器**（按价格优先、时间优先配对）→ 模拟市场经济。
  - 写一个 **计划分配器**（按物资平衡表、配额分配）→ 模拟计划经济。

将你的策略函数放入 `policies/` 目录，并注册到 Registry。

### 5. 运行并导出数据

```python
from core.simulator import Simulator
from policies.registry import Registry
import policies.demo_strategies as demo

# 1. 注册你的策略
reg = Registry()
reg.register("firm", demo.firm_strategy)
reg.register("household", demo.household_strategy)
reg.register("allocation", demo.demo_allocation)  # 这里替换你的核心制度

# 2. 启动模拟器
sim = Simulator("config/default.yaml", "seed_world.db")
sim.set_registry(reg)

# 3. 运行 30 个 Tick（相当于 30 个"月/季度"）
snapshots = sim.run(n_ticks=30)

# 4. 导出指标（基尼系数、失业率、恩格尔系数等）
import pandas as pd
df = pd.DataFrame(snapshots)
df.to_csv("experiment_results.csv", index=False)
```

## 模块机制与对应的经济学行为原理

对于经济学家而言，ESE 的每个模块都映射了现实经济运转中的特定制度或物理法则。以下是深入解释：

### 1. 数据实体层（Entities）：微观经济基础

- **Good（商品）**：定义了 `good_type`（食品/劳动力/资本品/原材料）和 `delivery_lag`（生产交付延迟）。

  > 经济行为：区分消费资料与生产资料，引入时滞（Time Lag）——这是奥地利学派经济学的核心，强调"计划赶不上变化"。

- **Firm（企业）**：持有现金、库存、产能和员工列表。

  > 经济行为：资产负债表约束。企业破产非瞬时，而是伴随复杂的清算顺序。

- **Household（家庭）**：持有现金、库存、就业状态和期望工资。

  > 经济行为：工人不仅是劳动力商品提供者，还是最终消费者。基尼系数基于家庭实际消费篮子计算，而非行业总产值。

### 2. 结算器（ClearingHouse）：硬性的"物理规则"（不可篡改）

这部分是 ESE 的灵魂，它不关心"计划"或"市场"，只负责执行不可违背的物理学与社会契约。

| 机制模块 | 工程实现细节 | 对应的经济学行为与原理 |
|---|---|---|
| 动态抵押品比率 | `calc_dynamic_collateral_ratio`：`base_ratio + (1 - 履约率) * 0.4` | 基于声誉的信贷约束（Reputation-based Collateral）。履约率高的主体（企业/家庭）只需缴纳少量保证金（10%）；赖账惯犯需缴纳高达 50% 的保证金。这模拟了现实中的信用评级体系和逆向选择风险。 |
| 双向冻结（双边保证金） | `freeze_collateral`：买卖双方同时被冻结资金。 | 合同法对等的履约担保。在现实衍生品交易中，买卖双方均需缴纳保证金。这避免了计划体制下"单方违约"的道德风险，确保市场或计划内的合同都具有真实信用支撑。 |
| All-or-Nothing 原子结算 | `settle_order`：只有库存且现金同时充足才算履约；一方不足即判定违约，抵押金全额赔偿对手方。 | 严格破产法与清算优先权。杜绝了"部分交付"的模糊空间。在宏观经济冲击下，流动性危机（现金不足）与供应危机（库存不足）是等价的，都会引发违约连锁反应（Contagion）。 |
| 破产清算（Liquidation） | `liquidate_firm`：优先清偿顺序为 ①拖欠工资 → ②欠税 → ③归零；库存折价变卖给政府。 | 工人主权与残余索取权。严格遵循现实法律（如《破产法》），劳动者报酬优先于股东权益。库存以市场最低价（`get_market_price_range.min`）甩卖，模拟了沉没成本和清算折价（Fire Sale）的残酷性。 |
| 级联违约（Cascade Default） | 破产时，该企业在 `pending_orders` 中的所有未交付订单全部标记 `DEFAULTED`。 | 系统性风险（Systemic Risk）。一家僵尸企业的倒下，会连带摧毁与其签有远期合约的交易对手方的预期收益，从而引发微观层面的信用崩塌。 |

### 3. 信息摩擦器（InformationFriction）：哈耶克的知识问题

**机制：** 在 Tick 末尾构建观测值（obs）时，对"他人"的财务报表（现金、库存）施加噪声（高斯噪声 / 向上瞒报 / 向下谎报）。

- 计划体制设定：`noise_type="none"`（计委直接调取企业真实账本，信息完美）。
- 市场体制设定：`noise_type="gaussian"`（企业只能通过打听、审计报表获取他人数据，存在统计误差）。

> 经济学原理：这是 **哈耶克（Hayek）** 在《知识在社会中的运用》中强调的分散知识（Dispersed Knowledge）问题。计划委员会即使拥有强大的计算能力，也无法获得与微观主体一样及时的局部信息。本模块量化了"信息收集成本"和"谎报瞒报"（`upward_bias` 模拟虚增利润，`downward_bias` 模拟藏匿收入）对宏观决策质量的破坏力。

### 4. 策略注册表与执行顺序（F→H→G）：时序博弈

**机制：** 每个 Tick 内，内核强制按 **企业（F）→ 家庭（H）→ 政府（G）** 的固定顺序依次调用策略。

> 经济学原理：这模拟了现实经济中的时序结构（Temporal Structure）。企业先决策（生产/定价），家庭后反应（消费/求职），政府最后兜底（征税/福利）。这避免了"同时决策"的不现实假设，使得价格粘性和预期调整在微观上得以体现。

### 5. 分配策略（AllocationPolicy）：制度的本质区别

**机制：** ClearingHouse 不包含任何配对逻辑。配对完全由用户编写的 AllocationPolicy 决定。

- **市场策略（Demo）**：遍历 supply_pool（卖方报价簿）和 demand_pool（买方询价簿），匹配 `supply.price <= demand.price`，取低价供给优先。
- **计划策略（供实验者编写）**：可无视价格信号，根据政府下达的"物资平衡表"直接向特定企业/家庭分配定额。

> 经济学原理：这就是 **兰格（Lange）与哈耶克（Hayek）大论战** 的工程落地。市场的本质不在于"如何算产量"，而在于"使用价格作为资源配置的信号"。通过替换这唯一的模块，实验者能在完全相同的初始条件下，对比"价格机制"与"配额机制"对总福利（基尼系数）的长期影响。

### 6. 账本（Ledger）与指标（Reporter）：宏观经济学的审计标准

**机制：** Ledger 记录每一笔订单的状态变更（Open → Allocated → Fulfilled/Defaulted），不可篡改。

> 经济行为：模拟现实中的 GAAP（公认会计准则） 审计轨迹。宏观指标（如 CPI 备选）必须基于历史交易数据回算，而非基于理论均衡值。

**指标计算：**

- **基尼系数（Gini）**：计入家庭现金 + 实物库存估值（用商品 `delivery_lag` 作为权重，模拟耐用品财富效应）。
- **恩格尔系数（Engel）**：仅统计近 30 个 Tick 的食品支出占比，反映居民实际生活水平。
- **失业率（Unemployment）**：基于 `is_employed` 标志位统计。

## 如何设计你的实验（制度对比三步法）

作为经济学家，你可以通过以下 3 个维度操控变量，观察结果差异：

1. **改变"信息体制"**（调整 `config/default.yaml` 中的 `noise_type`）：

   对比 `none`（完美信息） vs `gaussian`（市场噪音） vs `upward_bias`（企业财务造假）下，计划与市场的效率衰减曲线。

2. **改变"资源配置规则"**（重写 AllocationPolicy）：

   - 规则 A：价格优先（价低者得）—— 古典自由主义。
   - 规则 B：平均分配（每人固定配额）—— 绝对平均主义计划。
   - 规则 C：按历史产量比例分配 —— 投入产出表法（列昂惕夫）。

3. **改变"退出机制"**（修改 `ClearingHouse.liquidate_firm` 的清偿顺序——此处需源码级调整，默认已为最优破产法）：

   对比"刚性兑付"（政府兜底）与"残酷清算"（市场出清）对长期企业创新动力的抑制/促进作用。

## 输出指标解读

运行结束后，你将获得一个包含每个 Tick 快照的 CSV 文件：

| 字段 | 含义 | 经济学指示 |
|---|---|---|
| `gini` | 基尼系数 | 数值越高，贫富差距越大。市场体制通常上升，计划体制若配给精准可降低。 |
| `engel` | 恩格尔系数 | 数值越低，家庭越富裕。用于检验"生产是否真正改善了民生"。 |
| `unemployment` | 失业率 | 反映劳动力市场出清速度。计划体制可能强制就业（分母为0），市场体制存在自然失业。 |
| `active_firms` | 活跃企业数 | 反映市场新陈代谢速度。大量破产代表"创造性毁灭"，但也代表短期阵痛。 |

## 常见问题（FAQ for Economists）

**Q：为什么没有内置"投入产出表"（A 矩阵）？**

A：这是本项目的设计铁律。现实中的投入产出表是国家统计局根据抽样调查估计出来的，具有严重滞后性和误差。我们强迫实验者的算法只能通过历史交易账本（Ledger）自行回归推断 A 矩阵。这迫使"计划委员会"必须像现实中一样，为估计误差付出代价。

**Q：M0 货币总量恒定的意义？**

A：我们未引入央行印钞，是为了在 5 年（1825 Tick）尺度内剔除货币政策干扰，单纯观察资源配置制度的优劣。如果你需要模拟通胀，可在 Government 策略中自行实现货币增发（`money_supply` 字段已预留）。

**Q：为什么没有"内生技术创新"？**

A：这是当前版本的已知局限（见设计书附录）。我们建议通过外部设定（如每 500 Tick 随机激活一个企业的 capacity 升级）来模拟外生技术冲击，从而观察不同制度对新技术的采纳速度。
